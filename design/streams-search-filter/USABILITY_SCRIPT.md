Usability test script — Streams search/filter (internal, quick)

Goal
- Verify that users can find a paused stream using search + status filters in the new UI.

Participants
- 3 internal participants (product/ops/engineer mix recommended).

Task
- "Find a paused stream for recipient 'Acme Corp' and narrow results to USD asset."

Procedure
1. Show participant the desktop prototype (no guidance on where to look).
2. Ask them to complete the task; record time and whether they succeed without hints.
3. If stuck for 60s, prompt once: "What would you try next?" then allow another 60s.
4. After task, ask 3 short questions: "Was it clear how to search by recipient?", "Do the status chips make sense?", "Any confusion on mobile?"

Metrics
- Success (yes/no), time to complete, and one prioritized pain point per participant.

Deliverable
- Short summary: success rate, median time, 3 top pain points, and recommended fixes.
