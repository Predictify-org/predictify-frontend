Ready-to-send usability runner instructions

Overview
- Use these steps to quickly recruit and run the 3 internal usability sessions.

Which files to send
- Slack DM drafts: `USABILITY_DM_PERSONALIZED.md` or individual messages from `USABILITY_DM_MESSAGES.md`.
- Calendar invites (.ics): attach one of `INVITES/usability_slot_20260429_1100.ics`, `INVITES/usability_slot_20260429_1400.ics`, or `INVITES/usability_slot_20260430_0930.ics` depending on chosen slot.
- Calendar invite body: use `USABILITY_CAL_INVITE.md`.
- Prototype / review package: `design/streams-search-filter/review_package.zip` (or use the Figma file URL in the invite).

Steps to run
1. Pick a tester and preferred slot. (e.g., Ops -> 2026-04-29 11:00)
2. Paste the personalized DM from `USABILITY_DM_PERSONALIZED.md`, replace the placeholder name, and mention the slot.
3. Attach the matching `.ics` file from the `INVITES/` folder and send the DM or Slack message.
4. When tester confirms, send a calendar invite with `USABILITY_CAL_INVITE.md` content and attach the `.ics` if your calendar requires it.
5. During the session, run the script in `USABILITY_SCRIPT.md` and use `USABILITY_SESSION_CHECKLIST.md` as a checklist.
6. After each session, append the result line to `USABILITY_RESULTS.md` (create if missing) with participant id (P1), success, time (s), and top note.

Example result line for `USABILITY_RESULTS.md`
P1: Success=Yes, Time=38s, Note="Cleared filters not visible enough"

Notes
- Ask for consent to record. If declined, take notes manually.
- If a tester cannot join live, invite them to record a quick screencap and send notes.
