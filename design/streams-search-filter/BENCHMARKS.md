Benchmarks — fintech patterns for "many accounts" / streams

Purpose
- Capture 1–2 example patterns to compare search, filter chips, and mobile filter sheets.

Suggested examples (capture screenshots into Figma Benchmarks page)
- Mercury (accounts/transactions lists): compact list, clear search scoping, status labels.
- Brex / Ramp (treasury UI): multi-filter chips, persistent sort controls, clear empty-state guidance.

What to capture from each example
- Search placement and scoping (global vs scoped to recipient label).
- Chip design: single vs multi-select, visual affordance when active, overflow handling (scroll vs +more), touch targets.
- Mobile behavior: whether chips collapse into a sheet, or are horizontally scrollable.
- Sort control placement and phrasing (e.g., `Next payout`, `Newest`, `Largest amount`).

Notes for the designer
- Prefer clarity and predictability over visual novelty — users must trust money-adjacent UIs.
- Keep a consistent density between desktop and mobile; document row heights and truncation rules.
- Take screenshots and annotate in the Figma Benchmarks page with short notes.

Placeholders
- If you can't capture live screenshots, create mock snapshots in Figma following the notes above.
